# 相似历史日期查找功能说明

## 功能概述
根据当天的预判颜色组合（绿/红/黄），查找历史上最接近的日期，并展示它们的统计数据和走势图，帮助用户参考历史相似情况做出决策。

## 匹配算法

### 1. 匹配条件
- **必须包含当天所有存在的颜色**
  - 如果当天有绿色，历史日期也必须有绿色（数量可以不同）
  - 如果当天有红色，历史日期也必须有红色
  - 如果当天有黄色，历史日期也必须有黄色

### 2. 差值计算公式
```
差值 = |历史绿 - 当天绿| + |历史红 - 当天红| + |历史黄 - 当天黄|
```

### 3. 排序规则
按差值从小到大排序，差值越小越相似

## 示例计算

### 当天数据（2026-02-25）
```
绿色: 10根
红色: 1根
黄色: 1根
```

### 历史日期对比

#### 示例1：2026-02-12
```
绿色: 8根
红色: 2根
黄色: 2根

差值计算：
= |8 - 10| + |2 - 1| + |2 - 1|
= 2 + 1 + 1
= 4

✅ 满足条件（三色都有）
```

#### 示例2：2026-02-10
```
绿色: 7根
红色: 4根
黄色: 1根

差值计算：
= |7 - 10| + |4 - 1| + |1 - 1|
= 3 + 3 + 0
= 6

✅ 满足条件（三色都有）
```

#### 示例3：2026-02-04（假设）
```
绿色: 8根
红色: 4根
黄色: 0根  ← 没有黄色

❌ 不满足条件（缺少黄色）
```

### 最终结果
```
#1 最接近：2026-02-12 (差值=4)
#2 次接近：2026-02-10 (差值=6)
...
```

## 使用场景

```
当天预判：等待新低（绿10 红1 黄1）
找到相似日期：
- 2026-02-12：等待新低（绿8 红2 黄2）差值4
  → 最大涨幅 141.68%，可参考其走势

参考价值：

✅ 颜色组合相似

```

### 场景2：差值对比
```
差值越小 → 颜色组合越接近 → 参考价值越高

#1 差值=4：最接近，重点参考
#2 差值=6：次接近，辅助参考
#3 差值=7：一般接近，了解即可
```

### 场景3：统计数据分析
```
查看相似日期的：
1. 最低点：-14.45% → 最大回撤参考
2. 最高点：127.23% → 最大收益参考
3. 最大涨幅：141.68% → 日内振幅参考

决策支持：
- 如果历史相似日期都有大涨幅 → 今天可能也会有
- 如果历史相似日期最低点很低 → 注意风险控制
```

## 实现细节

### 1. 数据源
- **预判数据**：`data/daily_predictions/prediction_YYYY-MM-DD.json`
- **统计数据**：`data/coin_change_tracker/coin_change_YYYYMMDD.jsonl`

### 2. 数据计算流程
1. 读取当天预判文件，获取颜色统计
2. 扫描所有历史预判文件
3. 过滤出包含所有必需颜色的日期
4. 计算每个日期的差值
5. 按差值排序，取前N个
6. 读取每个日期的原始数据文件
7. 计算最低点、最高点、最大涨幅
8. 返回完整数据

### 3. 性能优化
- **后台加载**：不阻塞页面主流程
- **缓存控制**：使用 `no-store` 禁用缓存
- **异步加载**：懒加载（展开面板时才加载）
- **错误处理**：非关键功能失败不影响主功能
