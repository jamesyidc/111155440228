# Telegram通知策略文档

## 📋 概述

本文档详细说明日内模式监控器的Telegram通知策略，包括模式触发与开仓点的区别、通知发送次数规则、消息格式等。

## 🎯 核心概念

### 1. 模式触发 vs 开仓点

**重要区别：**
- **模式触发**：检测到特定的颜色序列模式（例如：🟢→🟡→🔴）
- **开仓点**：在模式触发后，27币涨幅之和达到特定阈值

**示例说明：**
```
时间       颜色      10分钟上涨占比    27币涨幅之和
04:50      🟢        67.59%           13.24%
05:00      🟡        48.61%           4.09%
05:10      🔴        42.59%          -1.39%   ← 模式触发时刻

状态：✅ 模式已触发（颜色序列符合）
      ⏳ 未达到开仓点（-1.39% < 65%阈值）
```

### 2. 为什么这样设计？

**问题背景：**
- 用户之前误解：认为65%阈值是指最后一根柱子的上涨占比
- 实际含义：65%阈值指的是**27个币种价格变化的总和**

**正确理解：**
1. 🟢→🟡→🔴 的颜色序列 = 模式已触发（这是一个预警信号）
2. 27币涨幅之和 ≥ 65% = 达到真正的做空点（这才是开仓时机）

## 📤 TG通知发送策略

### 双重分级通知规则

| 状态 | 说明 | TG发送次数 | 目的 |
|------|------|-----------|------|
| ⏳ **仅触发预警** | 模式已触发，但27币涨幅之和 < 阈值 | **1次** | 提醒用户跟踪，不骚扰 |
| ✅ **达到开仓点** | 模式触发 + 27币涨幅之和 ≥ 阈值 | **3次** | 重要交易信号，强调 |

### 阈值规则

**Pattern 1 (诱多等待新低)：**
- 大周期预判为"等待新低"时：阈值 = **65%**
- 大周期预判为"做空"或"观望"时：阈值 = **50%**

**Pattern 3 (筑底信号)：**
- 最后一根柱子上涨占比 < **10%**
- 且 当前涨跌幅总和 < **-50%**

**Pattern 4 (诱空信号)：**
- 中间所有红柱上涨占比 < **10%**

### Pattern 3: 筑底信号

消息格式与Pattern 1类似，会显示：
- 最后一根柱子上涨占比
- 当前涨跌幅总和
- 是否满足两个条件（<10% 且 <-50%）

### Pattern 4: 诱空信号

消息格式与Pattern 1类似，会显示：
- V型反转形态
- 中间红柱上涨占比是否 < 10%

## 🔄 实时监控流程

```
每10分钟执行一次（2:00-23:59 北京时间）
    ↓
获取最新10分钟柱子数据 + 27币原始记录
    ↓
检测所有模式（Pattern 1, 2, 3, 4）
    ↓
对每个检测到的模式：
    ├─ 评估是否满足开仓条件
    │   ├─ Pattern 1: 27币涨幅之和 ≥ 阈值？
    │   ├─ Pattern 3: 上涨占比<10% 且 总和<-50%？
    │   └─ Pattern 4: 中间红柱<10%？
    │
    ├─ 检查大周期过滤（小周期服从大周期）
    │
    ├─ 检查今天是否已通知过
    │
    └─ 发送TG通知
        ├─ satisfied=False → 发送1次（预警）
        └─ satisfied=True → 发送3次（开仓信号）
```

## ✅ 优点与改进

### 优点
1. **减少骚扰**：预警信号只发1次，不会频繁打扰
2. **强调重要性**：真正的开仓点发3次，确保用户不错过
3. **信息透明**：每条消息都显示具体数值和原因
4. **跟踪明确**：用户知道要关注什么指标（27币涨幅之和）

### 用户体验
- 🔔 收到1次TG → "好的，我知道有个模式触发了，我会关注27币涨幅之和"
- 🔔🔔🔔 收到3次TG → "重要！现在就是开仓点，立即操作！"
