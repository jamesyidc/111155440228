## 🔍 功能细节

### 通知发送机制
- **发送次数**: 每次标记发送5条相同的通知
- **发送间隔**: 每条通知间隔0.5秒
- **总耗时**: 约2-3秒

### 为什么发送5次？
1. ✅ **最高优先级**: 确保用户一定能看到
2. ✅ **强调重要性**: 多次通知强调信号强度
3. ✅ **防止错过**: 即使用户在忙，也不会错过
4. ✅ **提高响应速度**: 多次提醒促使快速决策

### 标记持久化
- 标记保存在浏览器的localStorage中
- 刷新页面后标记依然存在
- 可以随时修改标记类型
- 每次修改都会重新发送5次通知

## ⚠️ 常见问题

### Q1: 没有收到Telegram通知？
**检查项**:
1. Telegram配置文件是否存在：`config/configs/telegram_config.json`
2. bot_token和chat_id是否正确
3. 网络是否能访问Telegram API
4. 查看浏览器控制台是否有错误信息

**解决方法**:
```bash
# 检查配置文件
cat /home/user/webapp/config/configs/telegram_config.json

# 测试API
curl -X POST "http://localhost:9002/api/liquidation/mark-notify" \
  -H "Content-Type: application/json" \
  -d '{"mark_type": "long", "time": "2026-02-06T08:57:00+08:00", "amount": 15000}'
```

### Q2: 只收到部分通知（少于5条）？
**可能原因**:
- Telegram API限流
- 网络延迟或超时
- Bot Token配额限制

**解决方法**:
- 页面会显示实际发送成功的次数："Telegram通知已发送 X/5 次"
- 如果少于5次，可以重新点击标记再试一次

### Q3: 标记后页面无响应？
**可能原因**:
- 后端API异常
- 网络请求超时
- JavaScript错误

**解决方法**:
1. 打开浏览器开发者工具（F12）
2. 查看Console标签页的错误信息
3. 查看Network标签页的请求状态
4. 如果API返回错误，页面会显示具体的错误信息

### Q4: 标记刷新后消失了？
**原因**: localStorage被清除或浏览器隐私模式

**解决方法**:
- 不要使用浏览器的隐私/无痕模式
- 不要清除浏览器缓存
- 重新标记即可

### Q5: 能否撤销标记？
**答**: 可以，重新点击标记点，选择不同的类型即可覆盖。
- 每次修改都会重新发送5次Telegram通知
