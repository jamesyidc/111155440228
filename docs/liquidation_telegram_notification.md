# 爆仓月线图Telegram通知功能

## 📋 功能概述

为爆仓月线图添加了**最高优先级**的Telegram通知功能，当用户手动标记多空趋势转变时，系统会自动连续发送**5次**Telegram通知。

## 🎯 触发条件

当用户在爆仓月线图上点击标记点并设置多空类型时：
- 点击"多头爆仓（主升）"按钮 → 发送5次"多头爆仓→主升行情"通知
- 点击"空头爆仓（主跌）"按钮 → 发送5次"空头爆仓→主跌行情"通知

## ✨ 功能特点

### 1. 最高优先级
- 🚨 连续发送**5次**Telegram通知
- ⚡ 优先级标记为"最高"
- 🔥 信号强度标记为"极强"

### 2. 丰富的通知内容
- ⏰ 标记时间（当前时间）
- 📅 爆仓发生时间
- 💰 爆仓金额（亿/万美元）
- 📊 市场分析和趋势判断
- 🎯 交易建议和操作要点
- ⚠️ 风险提示

### 3. 智能判断
根据标记类型自动生成不同的分析：

#### 多头爆仓（主升行情）
- 趋势: 🟢 多头爆仓 → 主升行情
- 建议操作: 做多
- 原因: 大量多头爆仓后，空方力量耗尽，价格将反弹上涨

#### 空头爆仓（主跌行情）
- 趋势: 🔴 空头爆仓 → 主跌行情
- 建议操作: 做空
- 原因: 大量空头爆仓后，多方力量耗尽，价格将继续下跌

## 🔧 技术实现

### 后端API

**端点**: `POST /api/liquidation/mark-notify`

**请求参数**:
```json
{
  "mark_type": "long",  // 'long' 或 'short'
  "time": "2026-02-06T08:57:00+08:00",  // ISO 8601格式
  "amount": 15000  // 爆仓金额（万美元）
}
```

**响应示例**:
```json
{
  "success": true,
  "message": "Telegram通知已发送 5/5 次",
  "sent_count": 5,
  "mark_type": "long",
  "trend_name": "多头爆仓"
}
```

### 前端集成

修改了`setMarkType`函数，在用户点击标记按钮时：
1. 设置标记类型
2. 调用后端API发送Telegram通知
3. 显示成功提示（包含发送次数）
4. 保存标记到localStorage
5. 重新渲染图表

## 🎯 使用流程

### 1. 用户操作
1. 访问爆仓月线图页面：https://9002-imp6ky5dtwten0w001hfy-82b888ba.sandbox.novita.ai/liquidation-monthly
2. 点击图表上的红色标记点（爆仓金额 ≥ 1.45亿）
3. 在弹出的对话框中选择：
   - "多头爆仓（主升）" - 绿色按钮
   - "空头爆仓（主跌）" - 红色按钮

### 2. 系统响应
1. 立即向后端发送标记请求
2. 后端连续发送**5次**Telegram通知（间隔0.5秒）
3. 前端显示成功提示："✅ 标记已设置！Telegram通知已发送 5/5 次"
4. 标记保存到localStorage
5. 图表更新显示新的标记

### 3. 通知接收
- 用户会在Telegram收到**5条**相同的通知消息
- 通知间隔约0.5秒
- 确保不会错过重要的趋势转变信号

## ⚠️ 错误处理

### 前端错误处理
如果Telegram发送失败，前端会显示：
- ⚠️ 标记已设置，但Telegram通知发送失败: [错误信息]
- ⚠️ 标记已设置，但Telegram通知发送异常: [错误信息]

### 后端错误处理
可能的错误情况：
1. Telegram配置文件不存在
2. Telegram配置不完整（缺少bot_token或chat_id）
3. 网络请求失败
4. Telegram API返回错误

所有错误都会记录在控制台，并返回详细的错误信息。

## 💡 设计理念

### 为什么连续发送5次？
1. **最高优先级**: 爆仓趋势转变是重要的市场信号
2. **防止错过**: 多次发送确保用户一定能看到
3. **强调重要性**: 连续通知强调信号的重要程度
4. **提高响应速度**: 确保用户能及时做出交易决策

### 为什么不防重复？
1. **手动触发**: 每次标记都是用户主动操作
2. **历史回测**: 标记历史数据不需要防重复
3. **灵活性**: 用户可以反复标记和修改
