# 止损反手开单系统 - 使用说明

## ✅ 系统规则

### 🔑 执行反手的3个必要条件

止损反手监控器**只有在同时满足以下3个条件时**才会自动执行反手操作：

| 条件 | 说明 | 如何设置 |
|------|------|---------|
| 1️⃣ **启用开关** | `enabled = true` | 在前端页面打开"🔴 启用反手开单"开关 |
| 2️⃣ **触发权限** | `allow_trigger = true` | 首次启用自动为true；触发后需点击"🔄 重置"按钮 |
| 3️⃣ **选择策略** | `target_strategy_code` 不为空 | 在下拉菜单选择反手策略（涨幅前8/后8） |

### 🚫 不会执行反手的情况

| 状态 | 说明 | 日志提示 | 解决方法 |
|------|------|---------|---------|
| ⏸️ **未启用** | 开关关闭 | `⏸️ 配置未启用，跳过反手操作` | 打开前端启用开关 |
| 🔒 **已触发** | 触发后自动禁用 | `🔒 触发权限已禁用，需要手动重置` | 点击前端"重置"按钮 |
| ⚠️ **无策略** | 未选择反手策略 | `⚠️ 未设置目标策略，跳过反手操作` | 选择反手策略 |
| ⚪ **无配置** | 首次使用未创建配置 | `⚪ 无配置` | 访问前端页面自动创建 |

## 🎯 使用流程

### 第一步：启用反手开单

1. 访问 OKX Trading 页面
2. 找到 "🔄 止损反手开单" 面板
3. 选择对应的卡片：
   - **多单止损反手开空**（红色卡片）
   - **空单止损反手开多**（绿色卡片）
4. **打开启用开关**

```
🔴 启用反手开单  [   ON   ]
                  ^^^^^^^^
                  必须打开！
```

### 第二步：选择反手策略

在"执行策略"下拉菜单中选择：

**多单止损反手开空**（做空策略）：
- **STG_SHORT_TOP8**（涨幅前8名做空）
  - 适合：趋势反转，抓住反弹后的下跌
- **STG_SHORT_BOTTOM8**（涨幅后8名做空）
  - 适合：持续下跌，跟随弱势币

**空单止损反手开多**（做多策略）：
- **STG_LONG_TOP8**（涨幅前8名做多）
  - 适合：强势反弹，追涨
- **STG_LONG_BOTTOM8**（涨幅后8名做多，抄底）
  - 适合：超跌反弹，抄底

### 第三步：等待自动执行

```
暴跌预警触发 → 智能止损平仓 → 监控器检测到
                                    ↓
                            【检查3个条件】
                                    ↓
                    ✅ 3个条件都满足？
                            /          \
                         是             否
                         ↓              ↓
                   自动执行反手      跳过（日志提示原因）
                         ↓
                   TG通知 + 弹窗
                         ↓
                   触发权限禁用
```

### 第四步：触发后重置

反手执行一次后，触发权限自动禁用（防止重复）：

1. 卡片显示：🔴 **已触发，需重置**
2. 显示最后触发时间
3. 点击 **🔄 重置** 按钮
4. 确认重置
5. 权限恢复，可再次触发

## 🛠️ 故障排查

### 问题1：为什么没有执行反手？

**检查清单**：

```bash
# 1. 查看监控器日志
pm2 logs stoploss-reverse-monitor --lines 50 --nostream

# 2. 查看启动时的配置状态
# 找到 "📊 检查各账户止损反手配置状态..." 部分
# 确认你的账户显示 ✅ 而不是 ⏸️ 或其他

# 3. 检查配置文件
cat data/stoploss_reverse_orders/account_main_stoploss_reverse.jsonl

# 确认：
# - "enabled": true
# - "allow_trigger": true
# - "target_strategy_code": "STG_SHORT_TOP8" (或其他策略)
```

### 问题2：前端开关打开了，但日志显示"未启用"

**原因**：前端配置可能没有正确保存

**解决**：
1. 关闭开关
2. 选择策略
3. 重新打开开关
4. 等待几秒
5. 重启监控器：`pm2 restart stoploss-reverse-monitor`
6. 查看日志确认

### 问题3：触发后忘记重置，下次止损没有反手

**这是正常的防重复设计**

**解决**：
1. 访问前端页面
2. 找到对应的卡片
3. 点击"🔄 重置"按钮
4. 确认权限已恢复

### 问题4：如何测试反手是否会执行？

**不建议在生产环境测试**

**查看方法**：
```bash
# 查看监控器启动日志
pm2 logs stoploss-reverse-monitor --lines 100 --nostream | grep "检查各账户"

# 应该看到：
# • account_main: ✅ 多单止损反手开空 - 已启用 - 策略: STG_SHORT_TOP8
```

如果看到 ✅，说明配置正确，下次止损会自动执行反手。

## ⚠️ 重要提示

### 1. 监控器必须运行

```bash
# 检查监控器状态
pm2 status stoploss-reverse-monitor

# 如果显示 "stopped"，启动它：
pm2 start stoploss-reverse-monitor

# 如果显示 "online"，但长时间没反应，重启：
pm2 restart stoploss-reverse-monitor
```

### 2. 防重复机制

- ✅ 触发一次后自动禁用
- ✅ 防止连续反手导致损失放大
- ⚠️ 需要人工判断后手动重置
- 💡 这是安全设计，不是bug

### 3. 策略选择建议

| 市场情况 | 止损类型 | 推荐反手策略 | 原因 |
|---------|---------|-------------|------|
| 强势下跌 | 多单止损 | 涨幅前8名做空 | 反弹后继续下跌 |
| 持续下跌 | 多单止损 | 涨幅后8名做空 | 跟随弱势币 |
| 震荡市场 | 多单止损 | 涨幅前8名做空 | 高位反转 |
| 超跌反弹 | 空单止损 | 涨幅后8名做多 | 抄底机会 |

### 4. 日志保留

所有操作都有完整日志：

```
data/stoploss_reverse_orders/
├── logs/monitor_20260302.log          # 监控器日志
├── trigger_events/                     # 触发事件
│   └── account_main_trigger_events_20260302.jsonl
└── account_main_stoploss_reverse.jsonl # 配置文件
```
