# 23:50红柱显示问题调查报告

## 问题描述
用户反馈：10分钟上涨占比柱状图中，显示了一个23:50的红柱（14.80%），但该图表应该只显示0-2点的数据。

## 问题分析

### 1. 数据文件结构问题
经过调查发现，数据文件的命名和内容存在时间跨度问题：

**示例：coin_change_20260224.jsonl**
- 文件名表示：2026年2月24日
- 实际内容：**2月23日晚上20:31 → 2月24日晚上23:58**
- 小时范围：只包含 20, 21, 22, 23 点
- **缺失数据：没有0-2点的数据！**

**示例：coin_change_20260225.jsonl**
- 文件名表示：2026年2月25日
- 实际内容：**2月24日23:59 → 2月25日...**
- **包含前一天的23点数据！**

### 2. 过滤逻辑检查

原始代码（第7416-7418行）：
```javascript
const [hours, minutes] = timeStr.split(':').map(Number);
// 只保留0点和1点的数据（0:00-1:59）
return hours >= 0 && hours < 2;
```

**问题分析：**
- ✅ 逻辑本身是正确的：只保留 hours === 0 或 hours === 1
- ✅ 对于 "23:50:xx"，hours = 23，应该被过滤掉
- ❓ 但为什么图表还显示23:50的柱子？

### 3. 可能的原因

#### 原因A：数据文件确实包含23点数据
当查看2月25日的数据时，文件中包含：
```
2026-02-24 23:59:39
2026-02-25 00:01:04
2026-02-25 00:02:30
```

过滤器应该排除 `23:59:39` 这一条。

#### 原因B：图表渲染了非当天的数据
如果用户选择的日期（如2月25日）对应的数据文件没有足够的0-2点数据，可能导致：
1. `earlyMorningRawData` 为空或数据很少
2. `barData` 和 `timeLabels` 为空
3. 回退到 `barData.slice(0, 12)` 仍然为空
4. 图表可能显示缓存或错误的数据

#### 原因C：浏览器缓存
用户的浏览器可能缓存了旧版本的：
- HTML页面
- JavaScript代码
- 或者API响应数据

## 修复方案

### 修复1：增强过滤逻辑（已完成）
```javascript
// 🔥 关键修复：先过滤出0-2点的数据（必须是当天的0-2点，排除前一天的23点数据）
const earlyMorningRawData = data.filter(item => {
    // 获取完整的时间字符串（包含日期）
    const fullTimeStr = item.beijing_time || item.time || '';
    if (!fullTimeStr) return false;
    
    // 提取时间部分 (HH:MM:SS)
    const timeStr = fullTimeStr.includes(' ') ? fullTimeStr.split(' ')[1] : fullTimeStr;
    if (!timeStr) return false;
    
    const [hours, minutes] = timeStr.split(':').map(Number);
    
    // ⚠️ 严格检查：只保留小时为0或1的数据（0:00-1:59）
    // 排除小时为23的数据（即使在同一个文件中）
    const is0to2am = (hours === 0 || hours === 1);
    
    if (!is0to2am) {
        console.log(`🚫 排除非0-2点数据: ${fullTimeStr} (hour=${hours})`);
    }
    
    return is0to2am;
});
```

**改进点：**
1. ✅ 更严格的条件：`hours === 0 || hours === 1`（而不是 `>= 0 && < 2`）
2. ✅ 添加详细的调试日志，记录被排除的数据
3. ✅ 输出过滤后的前3条数据时间
