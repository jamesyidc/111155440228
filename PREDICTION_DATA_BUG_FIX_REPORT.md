## 错误原因分析

### 1. 数据来源问题
旧的预判文件可能是通过以下方式生成的：
- 使用了错误的API端点
- 使用了不同的数据计算逻辑
- 手动输入导致的人为错误

### 2. 最严重的问题类型
**"低吸"被错误标记为"等待新低"**（4个日期）：
- 2026-02-10
- 2026-02-12
- 2026-02-15
- 2026-02-20

这些日期的共同特点：
- 实际数据：**有绿有红无黄** → 应该是"低吸"（买入信号）
- 错误数据：添加了不存在的黄色柱子 → 变成"等待新低"（观望信号）

**影响**：用户在应该"低吸"（做多机会）的时候，系统错误地提示"等待新低"（观望），导致错过买入机会！

## 修复方案

### 创建重新生成脚本
`scripts/regenerate_all_predictions.py`

#### 脚本功能
1. 读取原始数据文件 `coin_change_YYYYMMDD.jsonl`
2. 筛选0-2点的数据
3. 按10分钟分组计算平均上涨占比
4. 判断颜色（绿>55%，黄45-55%，红<45%）
5. 根据颜色组合判断信号
6. 生成预判文件 `prediction_YYYY-MM-DD.json`

#### 判断逻辑（与监控脚本完全一致）
```python
# 情况1: 有绿+有红+无黄 → 低吸
if green > 0 and red > 0 and yellow == 0:
    signal = "低吸"
    
# 情况2: 有绿+有红+有黄 → 等待新低
elif green > 0 and red > 0 and yellow > 0:
    signal = "等待新低"
    
# ... 其他情况 ...
```
